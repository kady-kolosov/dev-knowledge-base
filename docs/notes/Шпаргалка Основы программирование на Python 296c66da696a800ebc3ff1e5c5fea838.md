# Знакомство с функциями

Функция — это именованный блок кода, который принимает параметры, возвращает значения, выполняется только при вызове и является объектом первого класса.

## Определение и вызов функции

Определение функции является этапом ее создания. Для определения функции необходимо создать:

- Заголовок — определяется ключевым словом `def`, за которым следует `имя_функции` ****в формате `snake_case`, затем в круглых скобках `()` перечисляются параметры функции, их может быть произвольное количество или не быть вовсе , и в самом конце строки после скобок `()` ставиться двоеточие **`:`**
- Тело — это блок кода, инструкции которого располагаются на новом уровне отступа, выполняются только при вызове функции.

```python
# Функция без параметров
def my_func():  # заголовок функции
   pass         # тело функции

```

Функция вызывается выражением `имя_функции()`.

```python
# Определяем функцию
def my_func():
    print('Функция вызвана')

# Вызываем функцию
my_func()  # ✅ ок: Функция вызвана
```

Вызвать функцию можно только после ее определения, иначе возникнет исключение `*NameError`.*

```python
# Вызываем функцию
my_func()  # ❌ NameError: name 'my_func' is not defined

# Опрежделяем функцию
def my_func():
    print('Функция вызвана')
```

# Позиционные и именованные аргументы

Процесс присваивания аргументов параметрам можно разделить на два типа: позиционный и именованный.

- Параметр функции — это переменная, которая используется в заголовке функции при ее определении.
- Аргумент функции — это значение, которое мы передается при вызове функции.

## Позиционные аргументы

При таком варианте передачи очень важна позиция определения самих параметров в функции. При позиционной передаче аргументов значения присваиваются параметрам функции в той же очередности, в какой параметры объявлены в функции.  Значит первый аргумент присваивается в первый параметр, второй аргумент — во второй параметр и т.д.

```python
def diff(x, y):
    return x - y

result = diff(10, 3)  # используем позиционные аргументы
print(result)         # 7
```

## Именованные аргументы

Использование именованных аргументов позволяет нарушать их позиционный порядок при вызове функции. Порядок именованных аргументов не имеет значения!

```python
def diff(x, y):
    return x - y

result = diff(x=10, y=3)  # используем именованные аргументы
print(result)  # 7

result = diff(y=3, x=10)  # меняем порядок именованных аргументы
print(result)             # 7
```

## Комбинация позиционных и именованных аргументов

При комбинированном варианте позиционные аргументы должны быть указаны до именованных!

```python
def example_combination(first, second, third):
    print(f'{first=}, {second=}, {third=}')

example_combination(1, 2, third=3)         # ✅ ок: first=1, second=2, third=3
example_combination(1, second=2, third=3)  # ✅ ок: first=1, second=2, third=3
example_combination(1, third=3, second=2)  # ✅ ок: first=1, second=2, third=3
example_combination(first=1, 2, 3)         # ❌ SyntaxError: positional argument follows keyword argument
```

# Обязательные и необязательные параметры

## Обязательные параметры

У обязательных параметров, при вызове функции мы обязаны передать ровно столько аргументов, сколько указано в заголовке функции, в обратном случае мы увидим исключение.

```python
def diff(x, y):
    return x - y
    

print(diff(10, 3))     # ✅: 7
print(diff(10))        # ❌ TypeError: diff() missing 1 required positional argument: 'y'
print(diff(10, 3, 7))  # ❌ TypeError: diff() takes 2 positional arguments but 3 were given
print(diff())          # ❌ TypeError: diff() missing 2 required positional arguments: 'x' and 'y'
```

## Необязательные параметры

Необязательные параметры — это параметры, которые принимают значение по умолчанию во время вызова функции. Чтобы задать значение параметра по умолчанию, в списке параметров функции достаточно после имени переменной указать оператор присваивания и нужное значение.

```python
def diff(x, y):
    return x - y

print(diff())          # ✅ ок: 7
print(diff(10))        # ✅ ок: 7
print(diff(10, 7))     # ✅ ок: 3
print(diff(10, y=7))   # ✅ ок: 3
print(diff(x=9, y=4))  # ✅ ок: 5
print(diff(1, 2, 3))   # ❌ TypeError: diff() takes from 0 to 2 positional arguments but 3 were given
```

## Комбинация обязательных и необязательных параметров

В определении  функции сперва описываются все обязательные параметры, только затем следуют необязательные параметры.

```python
print(make_circle(50, 50, 25))                                    # ✅ ок: x=50 y=50 radius=25 line_width=1 line_color='Blue' fill=True
print(make_circle(50, 50, 25, 2, 'Black'))                        # ✅ ок: x=50 y=50 radius=25 line_width=2 line_color='Black' fill=True
print(make_circle(50, 50, 25, line_width=2, line_color='Black'))  # ✅ ок: x=50 y=50 radius=25 line_width=2 line_color='Black' fill=True
print(make_circle(line_width=2, 50, 50, 25, line_color='Black'))  # ❌ SyntaxError: positional argument follows keyword argument
```

# Изменяемые значения по умолчанию: проблема и паттерн `None`

Не стоит использовать изменяемые объекты в качестве значений по умолчанию! Дело в том, что значение по умолчанию создаётся один раз — в момент определения функции, а не при каждом вызове.

```python
def append_to_list(value, my_list=[]):
    my_list.append(value)
    return my_list

# Тут передаем свой список [1, 2, 3] как аргумент my_list. Функция работает
# именно с этим отдельным списком, добавляет значение и возвращает его. Каждый
# вызов получает новый список, поэтому результат не зависит от предыдущего.
print(append_to_list(4, [1, 2, 3]))  # [1, 2, 3, 4]
print(append_to_list(5, [1, 2, 3]))  # [1, 2, 3, 5]

# Тут my_list не передан, значит используется значение по умолчанию [] из заголовка функции.
# Важный момент: значение по умолчанию создается один раз — в момент
# определения функции, а не при каждом вызове. Поэтому оба вызова работают с
# одним и тем же списком.
print(append_to_list(4))             # [4]
print(append_to_list(5))             # [4, 5]

# Нужно использовать None как маркер отсутствующего значения и создавать новый
# объект внутри функции
def append_to_list(value, my_list=None):
    if my_list is None:
        my_list = []
    my_list.append(value)
    print(f'{my_list=}, {id(my_list)=}')
    return my_list

append_to_list(40)                   # my_list=[40], id(my_list)=4379703552
append_to_list(50)                   # my_list=[50], id(my_list)=4381000640
```

# Чистая функция

Чистая функция — это функция, которая всегда возвращает один и тот же результат при одних и тех же входных данных и не имеет побочных эффектов.

1. Детерминированность —  это функция, которая возвращает один и тот же результат при одних и тех же входных данных. То есть функция не зависит от глобальных переменных или системного времени.
    
    ```python
    # ❌ Грязная: результат зависит от глобальной переменной
    rate = 0.2
    
    def get_tax(price):
        return price * rate
    
    print(get_tax(100))       # 20.0
    rate = 0.5                # меняем глобальную переменную
    print(get_tax(100))       # 50.0 — другой результат при одних и тех же аргументах
    
    # ✅ Чистая: возвращает один и тот же результат при одних и тех же входных данных
    def add(a, b):
        return a + b
    
    print(add(2, 5))          # 7
    print(add(2, 5))          # 7 — всегда одинаковый результат
    
    # ❌ Грязная: результат зависит от random или time
    from random import random
    
    def add_with_rand(x):
        return x + round(random(), 1)
    
    print(add_with_rand(10))  # 10.2
    print(add_with_rand(10))  # 10.8 — другой результат при одних и тех же аргументах
    
    # ✅ Чистая: возвращает один и тот же результат при одних и тех же входных данных
    def add(a, b):
        return a + b
    
    print(add(2, 5))          # 7
    print(add(2, 5))          # 7 — всегда одинаковый результат
    ```
    
2. Отсутствие побочного эффекта — это функция, которая не изменяет никакое внешнее состояние или данные за своими пределами. То есть не изменяет переданные аргументы и глобальные переменные, не взаимодействует с внешними системами.
    
    ```python
    # Не изменяет переданные аргументы, если они являются
    # изменяемыми, как списки, словари и множества. Не изменяет глобальные
    # переменные.
    # -----------------------------------------------------------------------------
    
    # ❌ Грязная: функция изменяет исходный список
    def add_element(data, element):
        data.append(element)
        return data
    
    my_list = [1, 2, 3]
    add_element(my_list, 4)
    print(my_list)            # [1, 2, 3, 4]
    add_element(my_list, 5)
    print(my_list)            # [1, 2, 3, 4, 5]
    
    # ✅ Чистая: создает копию списка, не изменяет исходный
    def add_element(data, element):
        data_copy = data.copy()
        data_copy.append(element)
        return data_copy
    
    my_list = [1, 2, 3]
    add_element(my_list, 4)
    print(my_list)            # [1, 2, 3]
    add_element(my_list, 5)
    print(my_list)            # [1, 2, 3]
    
    # ❌ Грязная: функция изменяет исходный список
    def get_squares(data):
        for i in range(len(data)):
            data[i] = data[i] ** 2
        return data
    
    my_list = [1, 2, 3]
    get_squares(my_list)
    print(my_list)            # [1, 4, 9]
    get_squares(my_list)
    
    print(my_list)            # [1, 16, 81]
    
    # ✅ Чистая: создает новый список, не изменяет исходный
    def get_squares(data):
        data_copy = [i ** 2 for i in data]
        return data_copy
    
    my_list = [1, 2, 3]
    get_squares(my_list)
    print(my_list)            # [1, 2, 3]
    get_squares(my_list)
    print(my_list)            # [1, 2, 3]
    
    # ❌ Грязная: использует изменяемое значение по умолчанию
    def append_to_list(value, data=[]):
        data.append(value)
        return data
    
    print(append_to_list(4))  # [4]
    print(append_to_list(5))  # [4, 5]
    
    # ✅ Чистая: создает новый список, используя None как маркер отсутствующего значения
    def append_to_list(value, my_list=None):
        if my_list is None:
            my_list = []
        my_list.append(value)
        return my_list
    
    print(append_to_list(4))  # [4]
    print(append_to_list(5))  # [5]
    
    # ❌ Грязная: функция изменяет глобальную переменную
    x = 5
    
    def get_square():
        global x
        x = x ** 2
        return x
    
    get_square()              # первый вызов меняет x
    print(f'{x=}')            # x=25
    get_square()              # второй вызов снова меняет x
    print(f'{x=}')            # x=625
    
    # ✅ Чистая: функция не изменяет глобальную переменную
    x = 5
    
    def double_value(value):
        return value * 2
    
    double_value(x)           # первый вызов не меняет x
    print(f'{x=}')            # x=5
    double_value(x)           # второй вызов снова меняет x
    print(f'{x=}')            # x=5
    
    # Не взаимодействует с внешними системами, а именно: не выполняет операции
    # ввода-вывода (чтение и запись, вывод в консоль), не обращается к БД и не делает
    # сетевых запросов.
    # -----------------------------------------------------------------------------
    
    # Чтение или запись:
    # Работа с файлами (чтение и запись) относится к вводу-выводу (I/O)
    # и зависит от внешнего ресурса — файловой системы.
    # Содержимое файла может меняться независимо от кода, поэтому такая функция
    # опирается не только на свои аргументы и имеет побочный эффект.
    # Чистая функция в этом случае должна получать уже готовые данные
    # и не выполнять чтение файла самостоятельно.
    # -----------------------------------------------------------------------------
    
    # ❌ Грязная: Выполняет операции чтения-записи
    def get_length(path):
        with open(path, 'r', encoding='utf-8') as file:
            text = file.readline().strip()
        return len(text)
    
    get_length('/Users/userkady/desktop/test.txt'))
    
    # ✅ Чистая: не выполняет операции чтения-записи
    def get_length(text):
        return len(text)
    
    file = open(r'/Users/userkady/desktop/test.txt', 'r', encoding='utf-8')
    file_readline = file.readline().strip()
    get_length(file_readline))
    
    # Вывод в консоль:
    # Вызов print относится к операциям вывода (I/O) и считается побочным эффектом.
    # Такая функция изменяет внешний мир (состояние стандартного потока вывода).
    # Чистая функция должна только формировать значение, а решение о выводе
    # (печать, логирование и т.д.) должно приниматься во внешнем коде.
    # -----------------------------------------------------------------------------
    
    # ❌ Грязная: выполняет вывод в консоль
    def print_greeting(name):
        print(f'Hi, {name}!')
    
    # ✅ Чистая: не выполняет вывод в консоль
    def format_greeting(name):
        return f'Hi, {name}!'
    ```
    

# Переменное кол-во аргументов: параметры `*args` `**kwargs`

## Получение позиционных аргументов в виде кортежа: параметр `*args`

`*args` в заголовке функции собирает все оставшиеся позиционные аргументы вызова в кортеж. Ели таких аргументов нет — `args` будет пустым кортежем.

В заголовке функции может быть только один такой параметр. Он должен стоять после всех позиционных параметров. Все параметры после `*args` становятся только именованными. Порядок параметров в определении функции выглядит следующим образом:

```jsx
def my_args(a, b, *args, c=123, d=456):
    print(a, b)              # обязательные позиционные
    print(args, type(args))  # остаток позиционных как кортеж
    print(c, d)              # keyword-only с дефолтами

my_args(1, 2, 3, 4, 5, c=1234, d=5678)
# 1 2
# (3, 4, 5) <class 'tuple'>
# 1234 5678
```

## Передача аргументов в форме списка и кортежа

- `*` при вызове функции — это распаковка итерируемого объекта в набор позиционных аргументов.
- Можно распаковывать несколько коллекций в одном вызове.
- Распаковку `*` можно смешивать с позиционными и именованными аргументами.
- Объект для `*` должен быть итерируемым, иначе будет TypeError.

```jsx
def my_args(*args):
    print(type(args))
    print(args)

my_args(1, 2, 3, *[4, 5, 6], *(7, 8, 9), 'abc')
# <class 'tuple'>
# (1, 2, 3, 4, 5, 6, 7, 8, 9, 'abc')
```

## Получение именованных аргументов в виде словаря: параметр `**kwargs`

Позиционные аргументы можно получать в виде *args, причем произвольное их количество. Такая возможность существует и для именованных аргументов. Только именованные аргументы получаются в виде словаря, что позволяет сохранить имена аргументов в ключах.

`**kwargs` в заголовке функции собирает все оставшиеся именованные аргументы вызова в словарь. Ели таких аргументов нет — `**kwargs` будет пустым словарем.

В заголовке функции может быть только один такой параметр. `**kwargs` пишется в самом конце, после последнего аргумента со значением по умолчанию. Можно комбинировать `*args` и `**kwargs`. Порядок параметров в определении функции выглядит следующим образом:

```python
def my_kwargs(a, b, *args, name='Gvido', age=50, **kwargs):
    print('a, b:', a, b)               # обязательные позиционные
    print('*args:', args, type(args))  # остаток позиционных как кортеж
    print('name, age:', name, age)     # keyword-only с дефолтами
    print('**kwargs:', kwargs)         # остаток именованных как словарь

my_kwargs(1, 2, 3, 4, 5, name='Timur', age=30, job='Teacher', language='Python')
# a, b: 1 2
# *args: (3, 4, 5) <class 'tuple'>
# name, age: Timur 30
# **kwargs: {'job': 'Teacher', 'language': 'Python'}
```

## Передача аргументов в форме словаря

- `**` при вызове функции — это распаковка словаря в набор именованных аргументов.
- Можно распаковывать несколько словарей в одном вызове.
- Распаковку `**` можно смешивать с позиционными и именованными аргументами.
- Ключи в словаре для `**` должны быть строками, иначе будет TypeError.

```python
def my_kwargs(*args, **kwargs):
    print(type(args), type(kwargs))
    print(args, kwargs)

my_kwargs(*[1, 2, 3], **{'name': 'Timur', 'age': 30, 'job': 'Teacher'})
# <class 'tuple'> <class 'dict'>
# (1, 2, 3) {'name': 'Timur', 'age': 30, 'job': 'Teacher'}
```

## Только именованные аргументы: символ `/` в сигнатуре

`/` — все, что до `/`, можно передавать только позиционно

```python
# Positional-only (до /)
def make_circle(x, y, radius, /, line_width=1, fill=True):
    pass

make_circle(50, 50, 25, 2, False)             # ✅ ок
make_circle(x=20, y=20, radius=10, 2, False)  # ❌ SyntaxError
make_circle(x=30, y=30, radius=15)            # ❌ TypeError
```

## Только позиционные аргументы: символ `*` в сигнатуре

`*` — все, что после `*`, можно передавать только по имени

```python
# Keyword-only (после *)
def make_circle(x, y, radius, *, line_width=1, fill=True):
    pass

make_circle(50, 50, 25, line_width=2, fill=False)             # ✅ ок
make_circle(x=20, y=20, radius=10, line_width=2, fill=False)  # ✅ ок
make_circle(50, 50, 25, 2, False)                             # ❌ TypeError
```

## Порядок параметров при их комбинации

```python
# [positional-only] /, [positional-or-keyword], *args, [keyword-only], **kwargs
def my_combo(a, b, /, c, d=10, *args, e, f=20, **kwargs):
    # a, b     — только позиционные (positional-only), до слеша
    # c, d=10  — позиционные или именованные, до звездочки
    # *args    — все оставшиеся позиционные в кортеже
    # e, f=20  — только именованные (keyword-only), т.к. идут после *args
    # **kwargs — все оставшиеся именованные в словаре
    print(f"{a=}, {b=}, {c=}, {d=}, {args=}, {e=}, {f=}, {kwargs=}")

my_combo(1, 2, 3, e=99)                            # ✅ ок: a=1, b=2, c=3, d=10, *args=(), e=99, f=20, kwargs={}
```
